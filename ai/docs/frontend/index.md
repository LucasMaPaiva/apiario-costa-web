# Frontend Docs Index

## Objetivo

Este diretorio e a base de referencia do frontend. O agent principal consulta este indice para decidir onde pesquisar conforme o tipo de tarefa.

## Como Usar

Se a tarefa for:

- arquitetura geral: leia `ai/rules/frontend-architecture.md`
- modularizacao: leia `ai/rules/frontend-modules.md`
- qualidade de codigo: leia `ai/rules/frontend-core.md`
- nomenclatura: leia `ai/rules/frontend-naming.md`
- componente: leia `ai/docs/frontend/components.md`
- hook: leia `ai/docs/frontend/hooks.md`
- service: leia `ai/docs/frontend/services.md`
- route: leia `ai/docs/frontend/routes.md`
- DTO: leia `ai/docs/frontend/dtos.md`
- model: leia `ai/docs/frontend/models.md`
- validator: leia `ai/docs/frontend/validators.md`
- pages e layouts: leia `ai/docs/frontend/pages.md`
- fluxo de feature: leia `ai/docs/frontend/feature-flow.md`

## Mapa Rapido de Pesquisa

### Nova feature

Leia nesta ordem:

1. `ai/docs/frontend/feature-flow.md`
2. `ai/docs/frontend/modules.md`
3. `pages/` relacionada
4. `modules/` relacionado
5. `ai/docs/frontend/checklists/new-feature.md`

### Integracao com API

Leia nesta ordem:

1. `ai/docs/frontend/routes.md`
2. `ai/docs/frontend/services.md`
3. `ai/docs/frontend/dtos.md`
4. `ai/docs/frontend/models.md`
5. hook relacionado

### Formulario

Leia nesta ordem:

1. `ai/docs/frontend/validators.md`
2. `ai/docs/frontend/hooks.md`
3. `ai/docs/frontend/components.md`
4. `ai/docs/frontend/checklists/new-form.md`

### Ajuste visual

Leia nesta ordem:

1. `layouts/` ou `pages/`
2. componente relacionado
3. modulo relacionado
