# Frontend Naming Conventions Guide

## Convencoes

- arquivos: `camelCase`
- hooks: `useSomething`
- classes: `PascalCase`
- funcoes: `camelCase`
- modulos: `snake_case`

## Regras de Nomeacao

- nome deve refletir intencao;
- componente deve ter nome semantico;
- hook deve comecar com `use`;
- service deve indicar acao;
- classes de model e DTO devem ser explicitas.
